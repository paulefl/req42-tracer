== What is req42-tracer?

`req42-tracer` is a CLI tool that makes requirements traceability visible and auditable.
It parses AsciiDoc documentation, connects requirements to architecture and tests, and validates ASPICE PAM 4.0 compliance — all from the command line.

== Prerequisites

* Go 1.22 or later (only needed to build from source)
* A project with AsciiDoc documentation (`.adoc` files)

== Installation

=== Pre-built binaries

Download the latest release from the https://github.com/paul-fleischmann-com/req42-tracer/releases[GitHub Releases page]:

[source,bash]
----
# Linux (amd64)
curl -L https://github.com/paul-fleischmann-com/req42-tracer/releases/latest/download/req42-tracer-linux-amd64 \
  -o req42-tracer && chmod +x req42-tracer

# macOS (Apple Silicon)
curl -L https://github.com/paul-fleischmann-com/req42-tracer/releases/latest/download/req42-tracer-darwin-arm64 \
  -o req42-tracer && chmod +x req42-tracer
----

=== Build from source

[source,bash]
----
git clone https://github.com/paul-fleischmann-com/req42-tracer.git
cd req42-tracer
go build -o req42-tracer ./cmd/req42-tracer
----

== Initialize a new project

The `init` command creates a ready-to-use project skeleton:

[source,bash]
----
req42-tracer init
----

Interactive prompts ask for project name, Go module path, and description.
For CI/CD or scripted setup, use non-interactive mode:

[source,bash]
----
req42-tracer init \
  --dir=./my-project \
  --name=MyProject \
  --module=github.com/myorg/myproject \
  --interactive=false
----

This creates:

----
my-project/
├── .req42.yaml                    # Configuration
├── docs/
│   ├── requirements/
│   │   └── req42.adoc             # Requirements document
│   └── arc42/
│       └── arc42.adoc             # Architecture document
├── architecture.jsonc             # Bausteinsicht model
└── reports/                       # Generated reports
----

== Your first traceability matrix

=== Step 1 — Add a requirement

In `docs/requirements/req42.adoc`:

[source,asciidoc]
----
[req,id=REQ-LOGIN-001,version=1,priority=high,aspice=SWE.1,status=approved]
== System shall authenticate users with username and password

The system shall provide a login form that accepts a username and password.
Credentials shall be validated against the user database.
----

=== Step 2 — Link it to architecture

In `docs/arc42/arc42.adoc`:

[source,asciidoc]
----
[arch,id=comp.auth,parent=comp.system,aspice=SWE.2,req=REQ-LOGIN-001,impl=internal/auth/handler.go]
== Authentication Component

Handles login, session management, and logout.
----

=== Step 3 — Run the trace command

[source,bash]
----
req42-tracer trace
----

Output:

----
TRACEABILITY MATRIX
================================================================================

Req: REQ-LOGIN-001 (high)
  System shall authenticate users with username and password
  ✅ Architecture: comp.auth (Authentication Component)
  ⚠️  Tests: no test specification found

Summary: 1 requirement(s), 1 covered by architecture (100.0%), 0 covered by tests (0.0%)
----

=== Step 4 — Validate

[source,bash]
----
req42-tracer validate
----

=== Step 5 — Live report

[source,bash]
----
req42-tracer watch --open
----

Opens an interactive HTML report in your browser that auto-refreshes when you save files.
